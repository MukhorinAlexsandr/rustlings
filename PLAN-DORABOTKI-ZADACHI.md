# План доработки: практические задачи по частям

Цель: добавить в проект **практические задачи по программированию** (как те, что мы решали: add_one, max_of_two, sum_array, find_max и т.д.) — **по каждой части учебника**, от **лёгких к сложным**. Логику и код существующих разделов не менять.

---

## 1. Общая идея

- Для **каждой из 11 частей** учебника завести набор задач.
- **Требование:** в каждой части — **минимум 10 задач**, из них **минимум 5 лёгких**.
- Задачи **решаются прямо в приложении**: встроенная проверка через компилятор на бэкенде. При верном решении — переход к следующей задаче; при неверном — объяснение и похожая задача, пока пользователь не решит.
- У каждой задачи: **уровень** (лёгкая / средняя / сложная), **формулировка**, **сигнатура функции**, **примеры ввода-вывода**, **связь с главами** (id глав), **объяснение при ошибке** (explanationWrong), **похожие задачи** (similarTaskIds).
- Задачи идут **от простых к сложным** внутри части и в целом по курсу.

## 2. Куда встроить задачи в приложении

Варианты (на выбор на этапе реализации):

| Вариант | Описание |
|--------|----------|
| **A** | Новый раздел в сайдбаре: «Практика» / «Задачи» — список частей, внутри части список задач с редактором кода и проверкой. |

Рекомендация: начать с **варианта B** (задачи привязаны к части) или **A** (отдельная вкладка «Практика»), чтобы не смешивать с текущими «первыми проектами».

## 2.1. Решение прямо в приложении: встроенный компилятор и поток задач

Задачи решаются **внутри приложения**, без перехода во внешнюю среду. Поведение такое.

### Встроенный компилятор

- В интерфейсе есть **редактор кода** (одна задача на экран).
- Пользователь пишет решение и нажимает **«Проверить»** (или «Запустить»).
- Код **компилируется и выполняется на бэкенде** (Rust: `rustc` или `cargo`, либо песочница). Результат проверки возвращается на фронт.
- Никакого перехода на внешний сайт или локальный терминал — всё происходит в приложении.

### Правильное решение

- Если решение **верное** (тесты прошли или вывод совпал с эталоном):
  - Показать сообщение об успехе (например: «Верно!»).
  - **Сразу перейти к следующей задаче** в рамках выбранной части (следующая по порядку в списке задач части).
  - Записать задачу как решённую (STATE, прогресс, XP при необходимости).

### Неправильное решение

- Если решение **неверное** (ошибка компиляции, паника или не прошли тесты):
  1. **Не** переходить к другой задаче.
  2. Показать **объяснение**:
     - при ошибке компиляции — вывод компилятора (stderr) и краткий разбор, что не так;
     - при неверном ответе — какой тест упал, ожидалось / получено, и короткая подсказка по идее решения.
  3. Выдать **новую задачу того же типа** (похожую на ту, что не решил):
     - либо из заранее заготовленного списка «похожих» задач (поле `similarTaskIds` у задачи),
     - либо одну из «вариантов» той же задачи (другие примеры/формулировка, но та же тема).
  4. Пользователь снова решает **в том же редакторе** (новая задача подставляется в интерфейс).
  5. Повторять шаги проверки → объяснение → похожая задача **до тех пор, пока пользователь не даст верное решение**. После верного решения — переход к следующей задаче по общему порядку, как в пункте «Правильное решение».

Итого: **решал в приложении → встроенная проверка (компилятор) → если верно — следующая задача; если неверно — объяснение + похожая задача, пока не решит.**

## 2.2. Похожие задачи (для цикла «пока не решишь»)

Чтобы при неверном решении подставлять задачу **похожую** на текущую:

- В JSON каждой задачи добавить поле **`similarTaskIds`**: массив id других задач той же части и той же (или на 1 ниже) сложности, на ту же тему.  
  Пример: у `part1_sum_array` в `similarTaskIds` могут быть `["part1_count_positive", "part1_count_zeros", "part1_find_max"]`.
- Либо для одной и той же задачи завести **несколько вариантов** (разные примеры/числа): поле **`variants`** — массив объектов с `examples` и `hint`. При неверном ответе показывать следующий вариант и его подсказку.
- При неверном решении: выбрать следующую задачу из `similarTaskIds` (или следующий вариант из `variants`) и отобразить её в редакторе; после верного решения вернуться к «основному» порядку списка задач части (следующая по id/порядку).

Это нужно явно прописать в формате задачи (п. 3) и в сценарии фронта/бэкенда (п. 5).

## 3. Формат одной задачи (для JSON/кода)

Пример структуры:

```json
{
  "id": "part1_add_one",
  "partId": 1,
  "difficulty": "easy",
  "title": "Прибавить 1",
  "description": "Функция принимает число x и возвращает x + 1.",
  "signature": "fn add_one(x: i32) -> i32",
  "examples": [
    { "input": "add_one(5)", "output": "6" },
    { "input": "add_one(-1)", "output": "0" }
  ],
  "chapters": [1, 2, 4],
  "template": "fn add_one(x: i32) -> i32 {\n    // твой код\n}",
  "explanationWrong": "Проверь: функция должна вернуть x + 1. Последнее выражение в функции без точки с запятой — это возвращаемое значение.",
  "similarTaskIds": ["part1_double", "part1_max_of_two"],
  "hints": ["В Rust возвращается последнее выражение без ;", "Можно просто написать x + 1"]
}
```

Поля для цикла «пока не решишь»:

| Поле | Назначение |
|------|------------|
| **explanationWrong** | Текст объяснения при неверном решении (или при ошибке компиляции — можно подставлять вывод компилятора + этот текст). |
| **similarTaskIds** | Массив id задач той же части, похожих по теме. При неверном решении показываем следующую из этого списка; когда пользователь решит её — возвращаемся к обычному порядку и переходим к следующей задаче в части. |
| **hints** | (Опционально.) Массив подсказок по одной; можно показывать по клику «Подсказка» или по очереди после каждой неудачи. |

Вариант с несколькими формулировками одной задачи:

```json
"variants": [
  { "examples": [{"input": "add_one(5)", "output": "6"}], "hint": "Верни x + 1." },
  { "examples": [{"input": "add_one(0)", "output": "1"}], "hint": "Не забудь про ноль." }
]
```

Для проверки: код отправляется на бэкенд, **компилируется и запускается** (Rust); результат тестов или stdout сравнивается с эталоном. Без встроенного запуска на сервере режим «решай в приложении» не работает.
