# План доработки RustLings — Web3, криптография, блокчейн

Предложения по добавлению тем Web3 в учебник. Rust широко используется в блокчейне: Solana, Polkadot/Substrate, Near. План разбит на этапы — от основ криптографии до смарт-контрактов.

---

## Общая логика

1. **Сначала криптография** — хеши, подписи. Без этого блокчейн непонятен.
2. **Потом концепции блокчейна** — теория, аналогии.
3. **Практика в Rust** — крейты sha2, ed25519-dalek, работа с ключами.
4. **Взаимодействие с блокчейном** — RPC, чтение данных.
5. **Смарт-контракты** — Solana/Anchor (опционально, сложнее).

Стиль: как для 12-летнего — аналогии, простые слова.

## Инструменты и крейты

### CLI (установка отдельно)

| Инструмент | Назначение | Установка |
|------------|------------|------------|
| **Solana CLI** | Ключи, баланс, airdrop, конфиг | `sh -c "$(curl -sSfL https://release.solana.com/stable/install)"` или `cargo install solana-cli` |
| **Anchor CLI** | Создание, сборка, деплой программ | `cargo install --git https://github.com/coral-xyz/anchor avm --locked` + `avm install latest` |

### Крейты Rust

| Крейт | Назначение |
|-------|------------|
| `sha2` | Хеширование SHA-256 |
| `ed25519-dalek` | Подписи Ed25519 |
| `bs58` | Кодирование Base58 (адреса) |
| `solana-client` | RPC-клиент Solana |
| `solana-sdk` | Создание транзакций, ключи |
| `anchor-lang` | Макросы и типы для Anchor-программ |
| `anchor-spl` | SPL-токены в Anchor |

## Риски и ограничения

1. **Сложность** — смарт-контракты и Anchor требуют понимания аккаунтов, CPI, rent. Для «12-летнего» стиля — упрощать.
2. **Актуальность** — Solana и крипто-экосистема меняются. Нужно проверять версии крейтов.
3. **Devnet** — для практики нужен Solana devnet. Главы про Solana CLI и Anchor включают установку и настройку.
4. **Безопасность** — важно подчеркнуть: приватные ключи не хранить в коде, не использовать mainnet для обучения.
