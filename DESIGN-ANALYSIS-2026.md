# Анализ дизайна RustLings — взгляд веб-дизайнера 2026

## Что уже хорошо

- **Цветовая система** — CSS-переменные, accent, семантика (success, warning, danger)
- **Темы** — dark/light с поддержкой `prefers-color-scheme`
- **Доступность** — skip-link, `prefers-reduced-motion`, aria-атрибуты
- **Шрифты** — Inter, Outfit, JetBrains Mono
- **PWA** — manifest, service worker
- **Адаптив** — breakpoints 900px, 600px

## Приоритет 1: Критичные доработки

### 1. Типографика для чтения

**Проблема:** В reader-body длинные тексты читаются без оптимизации под чтение.

**Рекомендации:**
- Ограничить ширину строки: `max-width: 65ch` для `.section-text`, `.analogy-text`
- Увеличить `line-height` для параграфов: `1.6` → `1.75`
- Увеличить размер шрифта: `14px` → `16px` для основного текста

### 2. Контраст и доступность (WCAG 2.2)

**Проблема:** `--text2` (#9898b8) и `--text3` (#5a5a7a) на тёмном фоне могут не проходить AA.

**Рекомендации:**
- Поднять контраст: text2 `#a8a8c8`, text3 `#7878a0`
- Усилить контраст для `.quest-num`, `.chapter-desc` в светлой теме

### 3. Фокус-состояния

**Проблема:** Фокус на кнопках и карточках не всегда заметен.

**Рекомендации:**
- Добавить `:focus-visible` с outline 2px accent
- Убрать outline только при `:focus:not(:focus-visible)`

### 4. Мобильная навигация

**Проблема:** Sidebar скрывается, но на мобильных нет нижней навигации — неудобно переключаться между разделами.

**Рекомендации:**
- Добавить bottom-nav на экранах < 900px для быстрого доступа к Учебникам / Урокам

## Приоритет 2: Важные улучшения

### 5. Иконки вместо эмодзи

**Проблема:** Эмодзи (🏠, 📚, 🎯) выглядят неоднородно в разных ОС и браузерах.

**Рекомендации:**
- Заменить на SVG-иконки (Lucide, Heroicons или свои)
- Или использовать единый набор emoji (Twemoji)

### 6. Анимации и переходы

**Проблема:** Переходы между страницами — мгновенные, нет ощущения «плавности».

**Рекомендации:**
- Добавить `view-transitions` (View Transitions API) для смены страниц
- Лёгкий fade-in для `.page-enter` (0.2s)
- Убрать `transform: translateY(-4px)` на hover для карточек — заменить на `box-shadow` или `scale(1.02)`

### 7. Hero-баннер

**Проблема:** На мобильных `hero-visual` скрыт — экран пустой.

**Рекомендации:**
- Оставить маленький краб (48–64px) на мобильных
- Уменьшить padding на мобильных
- Добавить градиентный фон вместо одного цвета

### 8. Кнопки и CTA

**Проблема:** Вторичные кнопки (`btn-secondary`) плохо отличаются от фона.

**Рекомендации:**
- Добавить `border: 1px solid var(--border2)` для вторичных
- Увеличить padding для touch: `min-height: 48px`.

### 9. Модальные окна

**Проблема:** Reader modal — много контента, мало «воздуха».

**Рекомендации:**
- Увеличить `padding` в reader-body
- Добавить scrollbar для длинных глав
- Кнопка «Пройти тест» — sticky в footer при скролле

## Приоритет 3: Полировка

### 10. Уведомления (toast)

**Сейчас:** Простой блок внизу.

**Рекомендации:**
- Позиция: top-right или top-center
- Анимация: slide-in + fade
- Тень и лёгкий blur

### 11. Карточки глав

**Рекомендации:**
- Добавить `:active` состояние (scale 0.98)
- Улучшить hover: `transition` 0.2s
- Добавить `will-change: transform` только на hover (для производительности)

### 12. Прогресс

**Рекомендации:**
- XP-бар: добавить градиентную анимацию при заполнении
- Уровень: лёгкий pulse при достижении нового уровня

### 13. Цветовая палитра

**Рекомендации:**
- Добавить `--accent-hover` для состояний hover
- Рассмотреть accent для «успех» (зелёный) вместо жёлтого в некоторых местах

### 14. Код-блоки

**Рекомендации:**
- Добавить кнопку «Копировать» в углу
- Синхронизировать тему highlight.js с темой приложения (light/dark)
