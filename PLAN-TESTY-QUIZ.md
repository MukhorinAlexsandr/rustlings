## Мнение

**Усложнение тестов по главам** — полезно: 3 простых вопроса слабо проверяют понимание. Добавление вопросов с кодом, поиском ошибок и более сложными вариантами ответов повысит ценность проверки.

**Супер-тест после части** — очень полезно: он закрепляет материал, показывает связи между главами и даёт ощущение завершённости части. Это логичная точка для награды (XP, достижение).

## Этап 1: Усложнение тестов по главам

### 1.1 Целевая структура

| Параметр | Было | Станет |
|----------|------|--------|
| Вопросов на главу | 3 | 4–5 |
| Вопросов с кодом | ~30% | 50–60% |
| Типы вопросов | Факт, вывод | Факт, вывод, ошибка, «что подставить» |
| Сложность вариантов | Простые | Правдоподобные отвлекающие варианты |

### 1.2 Типы вопросов

1. **Фактический** — «Что делает X?», «Чем отличается A от B?»
2. **Вывод кода** — «Что выведет/вернёт этот код?» (с блоком кода)
3. **Поиск ошибки** — «В какой строке ошибка?» или «Почему не компилируется?»
4. **Заполнение пропуска** — «Что подставить вместо ___?»
5. **Связь понятий** — «Какой тип использовать для X?»

### 1.3 Правила для вариантов ответов

- Всегда 4 варианта.
- Один верный, три правдоподобных.
- Не использовать «всё вышеперечисленное» или «ни один».
- Для кода — реальные варианты (другие ключевые слова, типы, методы).

### 1.4 Объём работ

- 98 глав × 1–2 новых вопроса ≈ 100–200 новых/переработанных вопросов.
- Рекомендуемый порядок: Часть I (главы 1–16), затем остальные части.

## Этап 2: Супер-тест после каждой части

### 2.1 Данные

**Новый файл:** `static/data/part-quizzes.json`

```json
[
  {
    "partId": 1,
    "title": "Супер-тест: Часть I — Основы",
    "desc": "Проверь знания по переменным, типам, функциям, циклам, структурам",
    "xp": 150,
    "minCorrect": 8,
    "questions": [
      {
        "question": "Что выведет этот код?",
        "code": "fn main() { let x = 5; let y = x + 1; println!(\"{}\", y); }",
        "options": ["5", "6", "Ошибка", "Ничего"],
        "correct": 1,
        "explanation": "x=5, y=6, println выводит 6"
      }
    ]
  }
]
```

### 2.3 Логика супер-теста

- Вопросы охватывают **все главы** части.
- Распределение: 1–2 вопроса на важную главу, 2–3 на ключевые.
- XP: 100–200 за часть (зависит от числа вопросов).
- `minCorrect` — минимум верных ответов для «зачёта» (например, 70%).
- Достижение: «Пройди супер-тест части I» и т.п.

### 2.4 UI

- После списка глав каждой части — кнопка «Супер-тест части I».
- Кнопка активна, когда все главы части прочитаны.
- Используется тот же overlay, что и для теста по главе, но с другими данными.
- В STATE: `completedPartQuizzes: [1, 2, 3, ...]`.
