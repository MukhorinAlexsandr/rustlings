## Этап 5: Разбить app.js на модули (продолжение)

**Цель:** Разделить монолитный app.js (~1150+ строк) на логические модули для удобства поддержки.

**Сложность:** Высокая  
**Влияние:** Высокий

### 5.1 Структура модулей

Предлагаемая структура:

```
static/
├── js/
│   ├── data.js        # Загрузка JSON, ACHIEVEMENTS (с condition)
│   ├── state.js       # STATE, loadState, saveState, XP, уровни, streak
│   ├── utils.js       # escHtml, sanitizeHtml, getTodayKey, getDailyProgress
│   ├── sidebar.js     # updateSidebar
│   ├── nav.js         # navigate, currentPage
│   ├── notify.js      # notify, fireConfetti
│   ├── views/
│   │   ├── dashboard.js
│   │   ├── why-rust.js
│   │   ├── textbook.js
│   │   ├── lessons.js
│   │   ├── first-project.js
│   │   └── achievements.js
│   ├── chapter.js     # Рендер главы, модалка, openChapter, closeChapterOverlay
│   ├── quiz.js        # Квиз по главе: renderChapterQuiz, quizState
│   ├── lesson.js      # Урок: renderLessonQuiz, lessonState, isTextbookPartComplete
│   ├── a11y.js        # Focus trap, Escape, overlay controls
│   └── main.js        # Инициализация, DOMContentLoaded, подключение всего
├── app.js             # Точка входа (или один бандл)
```

### 5.2 Порядок выполнения

| # | Задача | Описание |
|---|--------|----------|
| 5.2.1 | Создать js/ и базовые модули | data.js, state.js, utils.js — вынести и экспортировать |
| 5.2.2 | Вынести views | dashboard, why-rust, textbook, lessons, first-project, achievements |
| 5.2.3 | Вынести chapter, quiz, lesson | Модалки и логика квизов/уроков |
| 5.2.4 | Вынести sidebar, nav, notify, a11y | Оставшаяся логика |
| 5.2.5 | main.js и подключение | Собрать всё, обновить index.html | ✅ |
| 5.2.6 | Удалить старый app.js | После проверки работоспособности | ✅ |

### 5.3 Варианты сборки

- **Вариант A:** ES modules (`type="module"`) — без сборки, браузер сам подгружает
- **Вариант B:** Один бандл через скрипт (concat или esbuild) — меньше запросов

Рекомендация: начать с **Варианта A** (ES modules) для простоты.
